Cette fonction dite callback met à jour dynamiquement l'interface en fonction du mode sélectionné par l'utilisateur. Elle est appelée automatiquement lorsqu'un changement est détecté dans le menu principal.

En fonction du mode sélectionné :

  • Les conteneurs d’options (box_couleur ou box_robot) sont affichés ou masqués.
  • Le texte (choix) sélectionné est stocké dans la variable globale mode.
Parameters
widgetLe menu de sélection du mode
data(non utilisé ici)
  • Si aucun mode n’est sélectionné, on arrête le programme.
  • On met la valeur sélectionnée dans la variable globale
  • Libération mémoire
  • Si "Suivi de couleur", on affiche les paramètres de couleur.
  • On cache les paramètres du robot.
  • Sinon, on fait l'inverse.

Point d'entrée principal du programme GTK.

C'est dans le main() que l'on construit l'interface graphique, initialise et connecte les différents éléments de l'interface (menus, boutons, labels, etc...) à leurs callbacks respectifs.

En détail, on configure :

  • la fenêtre principale
  • les conteneurs, soient les "boîtes" qui détiennent les options propres à leurs cas (box_couleur, box_robot)
  • les menus déroulants
  • le bouton de commencement ("Lancer")

La fonction se termine en lançant la boucle principale de GTK pour rendre l'interface interactive.

Parameters
argcNombre d’arguments passés en ligne de commande
argvTableau des arguments
Returns
EXIT_SUCCESS si le programme se termine correctement
  • On initialise d'abord la librairie GTK.
  • On crée la fenêtre principale et ses propriétés.
  • On fait le choix d'un conteneur principal des éléments de la fenêtre en format vertical.
  • Les labels permettent de guider l'utilisateur à chaque étape.

Menu déroulant principal

  • Par défaut : Suivi Couleur
  • Déclenchement du callback quand l'utilisateur change de mode.

Création de la box pour les différentes couleurs du 1er mode

  • Par défaut : Rouge

Création de la box - Robot Police

  • Par défaut : Rouge
  • Par défaut : Port de masque

Création de bouton Lancer

  • Appel callback en cas de click.
  • On affiche tous les widgets de la fenêtre.
  • On masque la box du mode Robot Police qui sera activé que si le mode est choisi.

void on_lancer_button ( GtkButton *  button,
gpointer  user_data 
)

Callback déclenché quand l'utilisateur clique sur le bouton "Lancer".

Cette fonction récupère les paramètres sélectionnés dans l'interface utilisateur (en fonction du mode actif), les enregistre dans un fichier texte, puis exécute le programme externe correspondant :

  • En mode Suivi Couleur :
    • La couleur sélectionnée est sauvegardée dans le fichier couleur.txt.
    • Le programme camera_red_detect est lancé.
  • En mode Robot Police :
    • Les informations sur le tee-shirt et le port du masque sont sauvegardées dans config.txt.
    • Le programme robot_police_cam_mac est lancé.

Cette fonction permet donc de faire le lien entre l’interface GTK et les programmes externes.

Parameters
buttonLe bouton GTK qui a déclenché l'action (bouton "Lancer")
user_data(non utilisée ici)
